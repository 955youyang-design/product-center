## 1. 总览流程图

```mermaid
flowchart LR
  A["系统管理员维护店铺配置"] --> B["商品运营进入产品导入页"]
  B --> C["选择渠道并下载模板"]
  C --> D["整理导入数据并上传文件"]
  D --> E["执行校验"]
  E --> F{"校验是否通过"}
  F -- "否" --> G["查看错误报告并修正重传"]
  G --> D
  F -- "是" --> H["系统按商家编码建立关联"]
  H --> I["确认导入"]
  I --> J["写入产品、商品、库存、素材"]
  J --> K["导入记录页查看结果"]
  K --> L["渠道运营维护商品字段与详情页"]
  L --> M["库存运营维护库存并触发同步"]
```

## 2. 业务线拆分说明

- 导入业务线：承接模板下载、上传校验、商家编码关联和导入落库。
- 商品维护业务线：承接导入后商品的单条编辑、商品详情页管理、管理端映射结果读取和批量修改。
- 库存业务线：承接库存查看、人工调整和同步回写。
- 配置业务线：承接店铺配置维护；类目动态适配由中台管理端映射规则承接。

## 3. 导入业务线时序图

```mermaid
sequenceDiagram
  actor Admin as 系统管理员
  actor Operator as 商品运营
  participant ConfPage as 店铺配置详情
  participant ImportPage as 产品导入页
  participant ImportMech as 标准化导入机制
  participant TemplateAPI as 模板下载接口
  participant ImportAPI as 导入任务接口
  participant ImportObj as 导入任务对象
  participant ProdObj as 产品商品库存对象

  Admin->>ConfPage: 维护店铺规则配置
  ConfPage->>ImportMech: 发布店铺级规则
  Operator->>ImportPage: 选择渠道和店铺
  ImportPage->>TemplateAPI: 获取模板 profile 与模板文件
  TemplateAPI-->>ImportPage: 返回模板字段口径
  Operator->>ImportPage: 上传导入文件并执行校验
  ImportPage->>ImportAPI: 创建导入任务并发起校验
  ImportAPI->>ImportObj: 写入 draft 和 validating
  ImportMech->>ImportObj: 执行字段校验与商家编码关联
  ImportObj-->>ImportPage: 返回校验结果
  Operator->>ImportPage: 确认导入
  ImportPage->>ImportAPI: 执行导入
  ImportAPI->>ProdObj: 写入产品、渠道商品、库存、素材
  ImportAPI->>ImportObj: 回写 completed
```

## 4. 商品维护业务线时序图

```mermaid
sequenceDiagram
  actor ChannelOp as 渠道运营
  participant ItemList as 商品列表
  participant DetailList as 渠道商品详情页列表
  participant ItemDetail as 渠道商品详情
  participant DetailAPI as 商品详情页接口
  participant AssetList as 素材列表
  participant BatchPage as 批量修改工作台
  participant ItemMech as 商品维护与类目联动机制
  participant ItemAPI as 渠道商品接口
  participant RuleAPI as 管理端类目适配读取接口
  participant ItemObj as 渠道商品对象

  ChannelOp->>ItemList: 按渠道、店铺、状态筛选
  ItemList->>ItemAPI: 查询商品列表
  ItemAPI-->>ItemList: 返回命中结果
  ChannelOp->>ItemDetail: 打开单条商品详情
  ItemDetail->>RuleAPI: 获取店铺级管理端类目适配结果
  RuleAPI-->>ItemDetail: 返回管理端映射字段范围
  ChannelOp->>ItemDetail: 修改标题、价格、属性和履约信息
  ItemDetail->>ItemAPI: 保存商品详情
  ItemAPI->>ItemObj: 回写渠道商品
  ItemAPI->>ItemMech: 记录规则联动和审计信息
  ChannelOp->>DetailList: 进入渠道商品详情页列表并按商品筛选
  DetailList->>DetailAPI: 查询详情页列表与当前使用版本
  DetailAPI-->>DetailList: 返回详情页记录
  ChannelOp->>DetailList: 上传新详情页并设为当前使用
  DetailList->>DetailAPI: 创建详情页记录并关联渠道商品
  DetailAPI->>ItemObj: 回写当前使用详情页关联
  DetailAPI-->>AssetList: 详情素材沉淀到素材列表
  opt 批量修改
    ChannelOp->>BatchPage: 进入批量修改工作台
    BatchPage->>ItemAPI: 预览命中范围并执行批量修改
    ItemAPI->>ItemObj: 批量回写字段
  end
```

## 5. 库存业务线时序图

```mermaid
sequenceDiagram
  actor InvOp as 库存运营
  participant InvList as 库存列表
  participant InvMech as 库存同步与分配机制
  participant InvAPI as 库存接口
  participant SyncTask as 同步任务
  participant InvObj as 库存对象
  participant InvLog as 库存日志对象

  InvOp->>InvList: 按产品、渠道、店铺、SKU 筛选
  InvList->>InvAPI: 查询库存列表
  InvAPI-->>InvList: 返回库存快照
  InvOp->>InvList: 发起人工调整
  InvList->>InvAPI: 提交库存调整
  InvAPI->>InvObj: 更新库存快照
  InvAPI->>InvLog: 写入调整日志
  InvOp->>InvList: 触发库存同步
  InvList->>InvAPI: 发起同步请求
  InvAPI->>SyncTask: 创建同步任务
  SyncTask->>InvMech: 等待外部结果
  InvMech->>InvObj: 回写 success 或 failed
  InvObj-->>InvList: 展示同步状态
```

sequenceDiagram
