## 1. 角色分工

| 角色 | 核心职责 | 主要页面 | 关键结果 |
|---|---|---|---|
| 商品运营 | 维护产品主档、下载模板、执行导入、处理商家编码关联 | 产品列表、产品详情、产品导入、导入记录 | 产品主档和导入任务闭环 |
| 渠道运营 | 维护渠道商品售卖差异、处理类目与属性联动、管理商品详情页内容与版本、执行批量修改 | 商品列表、渠道商品详情页列表、渠道商品详情、批量修改 | 渠道商品售卖信息、详情页内容和素材引用关系正确 |
| 库存运营 | 维护库存总览、人工调整库存、发起同步、查看调整记录 | 库存列表、库存详情、库存调整记录 | 库存正确且同步状态可追踪 |
| 系统管理员 | 维护店铺配置与生效状态 | 店铺管理、店铺配置详情 | 默认类目、运费模板和发货时效等店铺配置稳定生效 |
| 平台管理员 | 管理客户、中台管理端用户和关键账号动作 | 客户管理、系统管理 | 客户主体和后台账号可控 |
| 运营管理员 | 维护客户、中台用户端用户、类目映射和属性值映射 | 客户管理、类目映射、属性值映射 | 客户隔离下的映射配置正确 |

## 2. 核心对象协同

| 对象 | 由谁主导 | 谁消费 | 协同说明 |
|---|---|---|---|
| 产品 | 商品运营 | 渠道运营、库存运营 | 产品是多渠道商品和库存归集中心。 |
| 渠道商品 | 渠道运营 | 库存运营 | 同一产品在不同渠道/店铺下可有多条渠道商品记录。 |
| 商品详情页 | 渠道运营 | 渠道运营、商品运营 | 同一产品在不同渠道会形成多个渠道商品，因此会有多套详情页；单个渠道商品下最多允许 3 个详情页，详情页内容可编辑，且可复用素材列表中的图片。 |
| 店铺配置 | 系统管理员 | 商品运营、渠道运营 | 配置决定导入模板字段口径和详情页管理端映射字段范围。 |
| 库存 | 库存运营 | 渠道运营 | 库存围绕渠道商品 SKU 维护，但在中台统一展示。 |
| 导入任务 | 商品运营 | 系统管理员、渠道运营 | 导入任务沉淀导入结果、错误原因和关联处理结论。 |
| 客户 | 平台管理员、运营管理员 | 中台用户端用户、映射管理 | 客户是中台管理端按商户隔离的主体。 |
| 类目映射 / 属性值映射 | 运营管理员 | 商品导入、商品维护、后续中台用户端 | 映射关系必须带 `customer_id`，不同客户可配置不同映射。 |
| 管理端用户 | 平台管理员、系统管理员 | 中台管理端 | 管理端用户不挂客户，用于后台管理权限控制。 |

## 3. 协同链路

1. 系统管理员先维护店铺配置。
2. 商品运营选择渠道商品导入或产品资料导入；渠道商品导入按单渠道、已授权店铺范围和单渠道类目下载模板，产品资料导入按产品类目下载模板，执行导入并确认商家编码或 ISBN 关联。
3. 渠道运营在渠道商品详情页维护售卖字段，在渠道商品详情页列表统一维护详情页内容、版本、当前使用状态和素材复用关系。
4. 库存运营在库存列表统一处理库存与同步结果。
5. 中台管理端由平台管理员和运营管理员维护客户、中台用户端用户、类目映射和属性值映射，后续作为中台用户端和独立用户系统的基础。
