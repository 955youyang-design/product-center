## 1. 页面目标

统一查看客户主体，并发起开客户、编辑、启停用和进入客户详情等操作。

## 2. 入口、出口与前置条件

| 项目 | 内容 |
|---|---|
| 入口 | 左侧菜单 `menu.mgmt.customer` |
| 出口 | 客户详情页 |
| 前置条件 | 登录且具备客户查看权限；开客户、编辑和启停用需要客户管理权限 |

## 3. 页面结构

| 区块 | 说明 | 当前重点 |
|---|---|---|
| 筛选区 | 按客户名称、客户编码、状态筛选 | 当前重点 |
| 统计区 | 展示客户总数、启用客户、停用客户 | 建议保留 |
| 表格区 | 展示客户基础字段和操作入口 | 当前重点 |
| 动作区 | 开客户、查看详情、启停用 | 当前重点 |

## 4. 主次操作

| 动作 | 触发条件 | 结果 |
|---|---|---|
| 开客户 | 点击主按钮 | 创建客户主体，默认进入启用或待配置状态 |
| 查看详情 | 点击客户名称或查看按钮 | 进入客户详情 |
| 编辑客户 | 有客户管理权限 | 进入客户详情基础信息区 |
| 启停用客户 | 有客户管理权限 | 修改客户状态并写入审计日志 |

## 5. 字段与校验

| 字段 | 编码 / 类型 | 规则 |
|---|---|---|
| 客户 ID | customer_id / string | 系统主键，只读 |
| 客户名称 | customer_name / string | 必填，列表展示和搜索 |
| 客户编码 | customer_code / string | 必填，建议唯一，不允许随意修改 |
| 客户状态 | customer_status / enum | enabled / disabled |
| 中台用户端用户数 | customer_user_count / number | 只读 |
| 属性值映射数 | attribute_mapping_count / number | 只读 |
| 最近更新时间 | updated_at / datetime | 只读 |

## 6. 状态与反馈

- 客户停用前需要二次确认，并提示该客户下中台用户端用户和属性值映射可能不可用；类目映射为平台全局规则，不进入客户停用影响范围。
- 客户编码重复时阻止创建。
- 无权限用户不能看到开客户和启停用入口。

## 7. 依赖对象与接口

| 类型 | 标识 | 作用 |
|---|---|---|
| 对象 | OBJ-MGMT-CUSTOMER | 客户主体 |
| 接口 | API-MGMT-CUSTOMER-LIST-GET | 查询客户列表 |
| 接口 | API-MGMT-CUSTOMER-CREATE-POST | 开客户 |
| 接口 | API-MGMT-CUSTOMER-STATUS-PUT | 启停用客户 |
