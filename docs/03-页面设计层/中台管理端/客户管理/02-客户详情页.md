## 1. 页面目标

维护客户基础信息，并在当前客户上下文内管理该客户的中台用户端用户。

## 2. 入口、出口与前置条件

| 项目 | 内容 |
|---|---|
| 入口 | 客户列表内容进入 |
| 出口 | 客户列表、类目映射列表、属性值映射列表 |
| 前置条件 | 必须携带 `customer_id`；用户具备客户查看权限 |

## 3. 页面结构

| 区块 | 说明 | 当前重点 |
|---|---|---|
| 头部摘要区 | 展示客户名称、编码、状态、最近更新时间 | 当前重点 |
| 基础信息区 | 编辑客户名称、备注、状态等信息 | 当前重点 |
| 中台用户端用户页签 | 查看、开通、编辑、启停用和重置中台用户端用户密码 | 当前重点 |
| 关联统计区 | 展示类目映射、属性值映射和中台用户端用户数量 | 建议保留 |

## 4. 主次操作

| 动作 | 触发条件 | 结果 |
|---|---|---|
| 保存客户信息 | 修改基础信息后提交 | 更新客户资料并写审计日志 |
| 开中台用户端用户 | 在中台用户端用户页签点击开用户 | 为当前客户创建中台用户端用户 |
| 编辑中台用户端用户 | 点击用户行操作 | 修改用户姓名、手机号、邮箱、角色和状态 |
| 重置中台用户端用户密码 | 有客户管理权限 | 生成临时密码或触发重置流程 |
| 跳转映射列表 | 点击关联统计 | 带 `customer_id` 进入对应映射列表 |

## 5. 字段与校验

| 字段 | 编码 / 类型 | 规则 |
|---|---|---|
| 客户 ID | customer_id / string | 必填，只读 |
| 客户名称 | customer_name / string | 必填 |
| 客户编码 | customer_code / string | 必填，只读或强管控修改 |
| 客户状态 | customer_status / enum | enabled / disabled |
| 中台用户端用户 ID | customer_user_id / string | 中台用户端用户主键 |
| 登录账号 | login_account / string | 必填，同一客户下唯一 |
| 用户姓名 | user_name / string | 必填 |
| 角色 | customer_user_role | enum | customer_admin / customer_operator / customer_readonly |
| 用户状态 | user_status / enum | enabled / disabled |

## 6. 状态与反馈

- 中台用户端用户必须归属当前客户，禁止跨客户编辑。
- 客户停用后，应阻止新增中台用户端用户，并提示当前客户不可用。
- 重置密码成功后只展示一次临时密码或重置结果提示。

## 7. 依赖对象与接口

| 类型 | 标识 | 作用 |
|---|---|---|
| 对象 | OBJ-MGMT-CUSTOMER | 客户主体 |
| 对象 | OBJ-MGMT-CUSTOMER-USER | 中台用户端用户 |
| 接口 | API-MGMT-CUSTOMER-DETAIL-GET | 查询客户详情 |
| 接口 | API-MGMT-CUSTOMER-DETAIL-PUT | 保存客户详情 |
| 接口 | API-MGMT-CUSTOMER-USER-LIST-GET | 查询客户下中台用户端用户 |

| 接口 | API-MGMT-CUSTOMER-USER-PUT | 保存中台用户端用户 |
| 接口 | API-MGMT-CUSTOMER-USER-PASSWORD-POST | 重置中台用户端用户密码 |
