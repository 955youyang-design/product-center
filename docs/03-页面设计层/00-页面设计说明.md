## 1. 当前页面设计层目标

- 把页面树中的页面节点和功能清单中的动作、规则能力落到可实现的前端页面与交互结构。
- 给前端明确每个页面的入口、布局、动作、字段和依赖接口。
- 明确哪些页面是V1.0，哪些仅为V1.0文档或历史兼容。

## 2. 当前页面组

| 页面组 | 当前状态 | 主要页面 | 非页面能力 |
|---|---|---|---|
| 产品管理 | V1.0 | 产品列表、产品详情、产品导入、导入记录、素材列表 | 新增产品、商家编码关联 |
| 商品管理 | V1.0 | 商品列表、渠道商品详情页列表、渠道商品详情 | 批量修改、读取中台管理端类目/属性映射结果 |
| 库存管理 | V1.0 | 库存列表 | 库存调整、库存同步；库存详情和记录为V1.0文档 |
| 配置管理 | V1.0 | 店铺管理、店铺配置详情 | 店铺配置维护 |
| 素材管理 | V1.0 | 素材详情 | 素材列表入口挂在产品管理下 |
| 中台管理端-客户管理 | V1.0 | 客户列表、客户详情 | 开客户、中台用户端用户管理、重置中台用户端用户密码 |
| 中台管理端-类目映射 | V1.0 | 类目映射列表、类目映射编辑页 | 全局类目映射、启停用映射、店铺级类目动态适配 |
| 中台管理端-属性值映射 | V1.0 | 属性值映射列表、属性值映射编辑页 | 客户隔离、依附类目映射、动态属性值适配 |
| 中台管理端-系统管理 | V1.0 | 管理端用户列表、管理端用户详情 | 开管理端用户、重置密码 |

## 3. 页面设计共性要求

- 列表页必须明确筛选区、表格区、分页或统计区和主操作区。
- 详情页必须明确保存动作、动态字段区、只读信息区和返回链路。
- 工作页必须明确任务状态、结果反馈和错误处理方式。
- 功能动作如果有独立工作台，也必须写清命中范围、预览和结果反馈。

## 5. 当前页面设计边界

- 页面文档只写页面节点本身，规则能力和动作承接在页面组或机制文档中补充。
- 当前V1.0重点是导入后商品、库存、配置的主流程闭环，不额外扩展独立发布类页面。
- 任何新增页面节点必须先回写页面树，再生成页面文档。
- 中台管理端当前为V1.0独立端，页面文档可作为研发前置输入，但实际进入开发前仍需确认是否纳入交付基线。
