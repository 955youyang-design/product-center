## 1. 页面目标

一个店铺对应一条配置记录。页面以表格汇总渠道店铺接入状态和店铺配置填写情况，帮助用户查看每个店铺是否已通过平台授权回调添加完成，以及是否已经维护导入商品所需的店铺配置。

## 2. 入口、出口与前置条件

| 项目 | 内容 |
|---|---|
| 入口 | 菜单 `menu.config`（店铺管理） |
| 出口 | 店铺配置详情 |
| 前置条件 | 配置查看权限 |

## 3. 页面结构

| 区块 | 说明 | 当前重点 |
|---|---|---|
| 筛选区 | 按渠道、店铺、配置状态筛选店铺 | 当前重点 |
| 列表区 | 表格展示店铺、平台、服务应用、订购状态、授权状态、配置填写、系统状态、默认类目、运费模板、发货时效、更新时间 | 当前重点 |
| 操作区 | 修改配置、复制设置、添加店铺 | 建议保留 |

## 4. 主次操作

| 动作 | 触发条件 | 结果 |
|---|---|---|
| 修改配置 | 点击店铺行操作 | 进入该店铺配置详情页 |
| 复制设置 | 点击复制设置 | 用当前店铺配置创建一条待授权的新店铺记录，不复制平台店铺 ID |
| 添加店铺 | 点击添加店铺 | 进入添加店铺流程，选择平台、生成携带参数的授权链接并等待平台回调 |

## 5. 字段与校验

| 字段 | 编码 / 类型 | 规则 |
|---|---|---|
| 渠道 | channel_code / 下拉 | 必填 |
| 店铺 | shop_name / 文本 | 平台授权后回填或用户补充 |
| 服务应用状态 | app_subscription_status / 标签 | 未订购、已订购 |
| 授权状态 | authorization_status / 标签 | 未授权、已授权、授权过期 |
| 授权有效期 | auth_expires_at / 日期 | 已授权店铺必填 |
| 默认类目 | default_category / 下拉 | 导入商品时的兜底归类设置，必填 |
| 系统自动状态 | config_status / 标签 | 由店铺授权与默认设置完成度自动生成，不提供单独设置 |

## 6. 状态与反馈

- 无配置时保留新增或复制入口。
- 一个平台店铺只对应一条店铺配置；平台店铺 ID 在系统内部作为唯一识别，不在列表明文展示。
- 平台店铺 ID 和授权凭证只在系统内部记录，列表不明文展示。
- 平台回调成功后，刷新列表即可看到新添加店铺。

## 7. 依赖对象与接口

| 类型 | 标识 | 作用 |
|---|---|---|
| 对象 | OBJ-SHOP-CONFIG | 店铺配置对象 |
| 接口 | API-CONF-LIST-GET | 配置列表查询 |

## 8. 范围边界

- 当前不拆系统级配置和渠道级配置独立页。
- 默认类目不是类目动态适配规则；类目映射和属性值映射仍在中台管理端维护。
