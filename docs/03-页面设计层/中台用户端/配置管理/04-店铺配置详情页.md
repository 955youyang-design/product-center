## 1. 页面目标

维护渠道店铺接入信息和店铺配置。添加店铺配置的第一步是添加店铺：用户先选择平台，并按页面提示在浏览器内提前登录对应平台账号；系统生成携带平台、店铺配置 ID、state 和回调地址的授权链接，同时进入回调监听状态。平台授权完成并回调成功后，系统内部记录店铺名称、平台店铺 ID、渠道、授权有效期等信息，其中平台店铺 ID 和授权凭证不向用户明文展示。店铺添加完成后，用户刷新列表即可看到新添加店铺，再继续维护店铺配置；店铺配置包括导入该店铺商品数据时使用的默认类目、默认上下架状态、运费模板和发货时效。

默认类目用于导入商品时兜底归类，不是店铺级类目动态适配规则。类目动态适配、类目映射和属性值映射不在本页维护，统一归中台管理端。

| 项目 | 内容 |
|---|---|
| 入口 | 店铺管理内容进入 |
| 出口 | 店铺管理、产品导入 |
| 前置条件 | 配置编辑权限 |

## 3. 页面结构

| 区块 | 说明 | 当前重点 |
|---|---|---|
| 第一步：添加店铺 | 选择平台、浏览器登录提示、授权链接、回调监听、平台授权状态、授权有效期、系统自动状态 | 当前重点 |
| 第二步：维护店铺配置 | 默认类目、运费模板、默认上下架状态、发货时效；未完成店铺添加前不可维护 | 当前重点 |
| 生效信息区 | 展示最近发布时间、发布人和生效说明 | 建议保留 |

## 4. 主次操作

| 动作 | 触发条件 | 结果 |
|---|---|---|
| 生成授权链接 | 用户选择平台并确认已登录平台 | 链接携带平台、店铺配置 ID、state、回调地址，系统进入监听回调状态 |
| 监听平台回调 | 授权链接发起后 | 收到回调后店铺添加完成，系统内部记录平台店铺 ID 和授权有效期 |
| 刷新查看店铺 | 回调成功后 | 返回列表后可看到新添加店铺 |
| 保存配置 | 店铺已授权且店铺配置完整 | 保存该店铺配置，系统自动计算配置状态 |

## 5. 字段与校验

| 字段 | 编码 / 类型 | 规则 |
|---|---|---|
| 店铺 | shop_code / 下拉 | 必填 |
| 服务应用 | app_service_name / 文本 | 由渠道决定 |
| 服务应用状态 | app_subscription_status / 标签 | 保存生效前必须已订购，不提供单独手动设置 |
| 授权状态 | authorization_status / 标签 | 保存生效前必须已授权，不提供单独手动设置 |
| 授权有效期 | auth_expires_at / 日期 | 已授权店铺必填 |
| 默认类目 | default_category / 下拉 | 导入商品时的兜底归类设置，必填 |
| 运费模板 | freight_templates / 下拉 | 从当前平台店铺可用运费模板集合中选择，必填 |
| 默认上下架状态 | default_shelf_status / 单选 | 上架或下架 |
| 发货时效 | default_delivery_sla / 下拉或输入 | 必填 |

## 6. 状态与反馈

- 一个平台店铺只对应一条店铺配置；复制设置只复用店铺配置，不复制平台店铺 ID 和授权凭证。
- 配置状态由系统根据店铺授权和默认设置完成度自动生成，不提供单独设置入口。
- 店铺配置保存完成后，系统自动将配置视为可用于后续导入任务。
- 渠道切换后需要重新订购服务应用并获取平台授权。
- 未监听到平台回调前，不允许维护默认类目、运费模板、默认上下架状态和发货时效。

## 8. 范围边界

- 当前不拆系统级配置、渠道级配置和模板配置独立页面。
- 当前页允许维护默认类目；不得新增或承接类目动态适配、类目映射、属性值映射配置入口，相关维护入口在中台管理端类目映射和属性值映射页面。
