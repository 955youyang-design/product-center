## 2. 当前页面脚手架建议

- 列表页统一采用：筛选区 + 指标区 + 表格区 + 操作区。
- 详情页统一采用：头部摘要 + 主表单区 + 关联信息区 + 页脚动作区。
- 工作页统一采用：说明区 + 任务输入区 + 校验区 + 结果区 + 确认区。
- 动作工作台统一采用：命中范围区 + 动作配置区 + 预览区 + 执行结果区。
- 原型应用允许提供右下角悬浮入口，在中台用户端和中台管理端之间切换；该入口仅用于原型演示，不作为业务菜单节点。

## 7. 路由调整要求

- 当前应保留：产品列表、产品详情、产品导入、导入记录、商品列表、渠道商品详情页列表、渠道商品详情、库存列表、配置列表、店铺配置详情。
- V1.0：素材列表、素材详情、库存详情、库存调整记录、批量修改工作台、新增产品。
- 历史兼容：历史发布记录占位页、系统配置详情、渠道配置详情、导入模板配置页、操作日志页。
- `overview/` 只保留重定向兼容，不再参与页面树、追踪矩阵和导航编排。
- `system-config/`、`publish/`、`operations/`、`reports/` 归为历史兼容目录；只有基线回写后才允许重新进入当前交付链路。
- 上游入口语义必须保持：产品导入和渠道商品详情页列表为V1.0二级菜单直达，批量修改为V1.0二级菜单直达，库存调整记录为V1.0二级菜单直达，新增产品为列表按钮进入。
- 渠道商品详情页列表承接全量详情页数据、内容编辑、当前版本切换和素材复用；渠道商品详情继续保持隐藏路由，只承接单商品售卖字段编辑。
- 前端实现必须直接体现“单个渠道商品最多 3 个详情页”的限制，并在列表和新建/上传动作中给出超限提示。
- 后续任何菜单显隐、排序、路径调整，都必须同步回写页面树、追踪矩阵、脚手架定义和 `product-platform.ts`，禁止只改前端路由而不改 docs。
