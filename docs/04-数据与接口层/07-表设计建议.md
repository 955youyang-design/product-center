## 2. 当前约束建议

- `shop_config` 必须通过唯一索引约束同版本唯一，同时通过业务逻辑保证单店铺单渠道单激活版本。
- `import_job_line` 需要支持大批量写入，建议按 `import_job_id` 分页查询，不建议全表扫描。
- `inventory_sync_task` 建议保留请求体和外部回执快照，便于排查同步失败。
- JSON 字段统一采用结构化 schema 管理，不要把动态字段直接拆成宽表列。
- `mgmt_customer_user`、`mgmt_category_mapping`、`mgmt_attribute_mapping` 必须以 `customer_id` 作为首要查询索引。
- 类目动态适配写入 `mgmt_category_mapping` 和 `mgmt_attribute_mapping`，不得再新增中台用户端类目动态适配维护表。
- `mgmt_admin_user` 不设置 `customer_id` 字段，避免和中台用户端用户模型混用。

## 3. 清理与归档建议

- `import_job`、`import_job_line`、`audit_log` 建议按时间维度做归档策略。
- `inventory_log` 和 `inventory_sync_task` 不建议物理删除，至少保留用于审计和故障排查的窗口期。
