## 1. 端定位

中台管理端是一个**独立于当前中台用户端**的新端，用于承接客户主体管理、按客户隔离的渠道映射关系管理，以及两类账号管理能力。

它与当前中台用户端的分工如下：

| 端 | 主要使用角色 | 核心职责 |
|---|---|---|
| 中台用户端 | 商品运营、渠道运营、库存运营、系统管理员 | 管理产品、商品、库存、店铺配置等业务数据，并读取管理端映射结果 |
| 中台管理端 | 平台管理员、运营管理员、系统管理员 | 管理客户主体、店铺级类目动态适配、类目映射、属性值映射、中台用户端用户和管理端用户 |

当前端的设计要求：

1. 复用当前项目既有的页面设计规范。
2. 保持独立菜单树，不挂在现有商品中台菜单下。
3. 后续可以演进为独立的用户系统与独立前端应用。

## 2. 使用角色

| 角色 | 核心目标 | 核心入口 | 关键产出 |
|---|---|---|---|
| 平台管理员 | 管理客户、映射关系和后台账号 | 客户管理、类目映射、属性值映射、系统管理 | 客户主体、映射规则、后台账号 |
| 运营管理员 | 维护客户和映射规则 | 客户管理、类目映射、属性值映射 | 可用的客户与映射配置 |
| 系统管理员 | 管理后台账号与系统侧权限边界 | 系统管理 | 管理端用户账号、账号状态 |

## 3. 核心场景

| 场景 | 参与角色 | 场景说明 | 主要结果 |
|---|---|---|---|
| 客户主体建立 | 平台管理员、运营管理员 | 创建客户、维护客户基础信息、启停用客户 | 客户主体可用，作为商户用户和映射隔离基础 |
| 中台用户端用户管理 | 平台管理员、运营管理员 | 在客户详情下管理某个客户的中台用户端用户 | 中台用户端用户按客户归属清晰可控 |
| 类目映射维护 | 平台管理员、运营管理员 | 按客户维护渠道类目到商品中台标准类目的映射，并形成店铺级类目动态适配规则来源 | 类目映射规则可配置，用户端只读取结果 |
| 属性值映射维护 | 平台管理员、运营管理员 | 按客户维护渠道属性值到中台标准值的映射，参与商品详情动态字段和值域适配 | 属性值映射规则可配置，用户端只读取结果 |
| 管理端用户管理 | 平台管理员、系统管理员 | 管理中台管理端自己的后台用户 | 后台账号可用且状态可控 |

## 10. 核心对象与数据隔离

### 10.1 对象清单

| 对象 | 说明 | 主归属 |
|---|---|---|
| 客户 | 中台用户端用户与映射关系的统一隔离主体 | 管理端 |
| 中台用户端用户 | 某个客户下的业务账号 | 客户 |
| 管理端用户 | 中台管理端自身后台账号 | 管理端 |
| 类目映射 | 某客户下渠道类目到中台类目的映射关系 | 客户 |
| 属性值映射 | 某客户下渠道属性值到中台标准值的映射关系 | 客户 |

### 10.2 关系表

| 关系 | 说明 |
|---|---|
| 客户 1:N 中台用户端用户 | 中台用户端用户必须归属某个客户 |
| 客户 1:N 类目映射 | 类目映射按客户隔离 |
| 客户 1:N 属性值映射 | 属性值映射按客户隔离 |
| 类目映射 1:N 属性值映射 | 属性值映射依附于类目映射上下文 |
| 管理端用户与客户无归属关系 | 管理端用户独立于客户主体 |

### 10.3 隔离原则

1. 类目映射和属性值映射必须带 `customer_id`。
2. 同一渠道类目在不同客户下允许映射到不同中台类目。
3. 同一渠道属性值在不同客户下允许映射到不同标准值。
4. 中台用户端用户不能脱离客户主体独立存在。
5. 管理端用户不参与客户隔离。

## 14. 评审重点

在进入 baseline 回写和页面设计层细化之前，需要优先确认：

1. 独立管理端的正式名称。
2. 根路径是否固定采用 `/management`。
3. 客户停用时，是否连带停用该客户下中台用户端用户与映射关系。
4. 属性值映射是否必须依赖类目映射存在。
5. 管理端用户是否在 V1 暴露固定角色字段。

## 15. 结论

本设计建议新增一个**中台管理端**，作为独立端承接客户管理、按客户隔离的映射管理，以及两类账号管理能力。

V1 菜单结构采用：

1. 客户管理
2. 类目映射
3. 属性值映射
4. 系统管理

其中：

- 中台用户端用户管理挂在客户详情下承接，不单独升为一级菜单。
- 类目映射和属性值映射全部按客户隔离。
- 管理端用户独立于客户主体管理。
