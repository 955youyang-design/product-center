## 1. 机制目标

统一定义中台用户端店铺配置保存后的生效时点和版本切换规则。类目动态适配由中台管理端类目映射和属性值映射维护，不在本机制写入。

## 2. 触发条件

- 系统管理员在店铺配置详情页保存配置
- 商品运营在产品导入页下载模板

## 3. 生效规则

1. 店铺配置以版本化方式保存，状态分为 `draft`、`active`、`inactive`。
2. 同一店铺同一渠道同一配置类型，在任一时点只允许一个 `active` 版本。
3. 新保存的配置先进入 `draft`，经确认后切换为 `active`；原 `active` 版本自动转为 `inactive`。
4. 模板下载始终读取当前 `active` 配置版本。
5. 已创建但未执行的导入任务，默认使用创建任务时快照的配置版本；是否允许追随最新版本，以 `ISSUE-003` 最终结论为准。

## 6. 异常与回退

- 同一店铺存在两个 `active` 版本时，禁止继续发布新版本并要求人工处理。
- 店铺配置拉取失败时，导入页禁止继续执行下载模板或校验动作。
- 配置保存成功但版本切换失败时，必须整体回滚，不允许出现“保存成功但未生效”的脏状态。
