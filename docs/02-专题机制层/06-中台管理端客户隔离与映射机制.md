## 1. 机制目标

中台管理端以客户作为数据隔离主体，确保中台用户端用户、类目映射、属性值映射和店铺级类目动态适配规则都在客户上下文内管理，避免不同客户之间共享、串改或误用映射规则。中台用户端只消费这些映射结果，不维护规则本身。

## 2. 隔离对象

| 对象 | 是否必须带 customer_id | 说明 |
|---|---|---|
| 客户 | 否 | 客户自身是隔离主体。 |
| 中台用户端用户 | 是 | 必须归属某个客户。 |
| 类目映射 | 是 | 不同客户可对同一渠道类目配置不同中台类目。 |
| 属性值映射 | 是 | 不同客户可对同一渠道属性值配置不同标准值。 |
| 店铺级类目动态适配 | 是 | 由类目映射和属性值映射组合生成，按客户、渠道、店铺和类目供中台用户端读取。 |
| 管理端用户 | 否 | 管理端用户是后台账号，不挂客户。 |

## 3. 映射规则

| 规则 | 说明 |
|---|---|
| 类目映射唯一性 | 同一 `customer_id + channel_code + channel_category_id` 建议只允许一条 active 映射。 |
| 属性值映射唯一性 | 同一 `customer_id + category_mapping_id + channel_attr_key + channel_attr_value` 建议只允许一条 active 映射。 |
| 类目映射依赖 | 属性值映射建议依附有效类目映射；类目映射停用后，其下属性值映射不得继续作为 active 规则使用。 |
| 动态适配输出 | 中台管理端根据有效类目映射和属性值映射输出可维护字段范围、属性值范围和标准值映射结果。 |
| 用户端消费边界 | 中台用户端商品详情、导入校验和模板字段只读取动态适配结果，不允许在用户端新增或编辑映射规则。 |
| 客户停用影响 | 客户停用后，新增中台用户端用户、类目映射和属性值映射应被阻止。 |

## 4. 权限边界

| 操作 | 权限要求 | 审计要求 |
|---|---|---|
| 开客户 / 编辑客户 / 启停用客户 | 客户管理权限 | 必须记录客户 ID、动作、操作人、时间 |
| 开中台用户端用户 / 重置密码 | 客户管理权限 | 必须记录 customer_id 和 customer_user_id |
| 新增 / 编辑 / 启停用映射 | 映射管理权限 | 必须记录 customer_id、mapping_id 和动作 |
| 发布 / 刷新动态适配结果 | 映射管理权限 | 必须记录 customer_id、channel_code、shop_code、category_mapping_id 和动作 |
| 开管理端用户 / 重置密码 | 系统管理权限 | 必须记录 admin_user_id，不记录 customer_id |

## 5. 前端与接口约束

- 客户详情页是中台用户端用户管理的唯一承接页面。
- 类目映射和属性值映射列表必须支持按客户筛选。
- 店铺级类目动态适配规则必须由中台管理端维护；中台用户端只能通过只读接口消费映射结果。
- 所有映射写接口必须在服务端校验 `customer_id`，不能仅依赖前端传参。
- 管理端用户接口不得提供客户筛选或客户归属字段。
