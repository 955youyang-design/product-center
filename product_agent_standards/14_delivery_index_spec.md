# 项目交付索引与当前有效版本清单规范

| 字段 | 内容 |
|---|---|
| 文档名称 | 14_delivery_index_spec.md |
| 文档类型 | 项目交付索引规范 |
| 适用范围 | 项目实例文档定位、当前有效版识别、归档后检索 |
| 适用角色 | 产品经理、产品 Agent、研发、测试 |
| 当前版本 | V1 |
| 维护人 | 产品 Agent 标准维护者 |
| 更新时间 | 2026-04-17 |

## 1. 文档目标

本文件定义一份项目级索引，用于回答：

- 当前项目实际交付了哪些文档
- 哪些是当前有效版
- 哪些已被替代
- 哪些已归档

## 2. 最小字段

建议至少包含：

| artifact_id | artifact_name | file_path | artifact_type | current_version | doc_status | baseline_version | supersedes | archive_path | owner |
|---|---|---|---|---|---|---|---|---|---|

## 3. 必须纳入索引的内容

至少应纳入：

- 三份输入基线
- 00-05 分层文档
- 问题清单与确认记录
- 需求追踪矩阵
- 评审签收记录

## 4. 更新规则

以下情况应更新项目交付索引：

- 新增正式文档
- 文档版本升级
- 当前有效文档被替代
- 文档归档
- 评审签收状态变化

## 4.1 复核频率建议

建议至少在以下时点复核项目交付索引：

- 每轮正式迭代结束后
- 每次评审签收完成后
- 每次归档或回滚后

如果索引长期未更新，不应默认视为仍然有效。

## 5. 当前有效版判断规则

当前有效版应至少满足：

- `doc_status` 为 `approved` 或 `locked`
- 未被更新版本替代
- 在项目交付索引中被标记为当前有效

## 6. 输出位置建议

建议放在：

`docs/00-总览层/07-项目交付索引与当前有效版本清单.md`

## 7. 完成标准

一份合格的项目交付索引，至少应满足：

- 新成员能快速找到权威文档
- 归档与现行版本不混淆
- 每份正式文档都能被定位到
