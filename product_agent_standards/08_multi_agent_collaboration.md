## 1. 文档目标

本文件定义多个产品经理 Agent 并行工作时的拆分原则、产出边界和汇总规则，避免多人协同时口径漂移。

## 2. 角色定义

### 2.1 主 Agent

主 Agent 负责：

- 与产品进行主对话
- 收敛范围与假设
- 建立三份输入基线
- 拆分任务给子 Agent
- 汇总各子 Agent 产出
- 解决冲突并形成最终版本

### 2.2 子 Agent

子 Agent 负责：

- 在被分配边界内整理文档
- 不越权新增范围
- 把输出回挂到统一基线
- 显式标记假设与待确认项

## 3. 推荐拆分方式

优先按以下维度拆分：

1. 先按层级拆分
2. 再按端拆分
3. 再按业务线或页面组拆分

推荐任务包示例：

- Agent A：范围收敛与输入基线
- Agent B：00-总览层
- Agent C：01-端产品层
- Agent D：02-专题机制层
- Agent E：03-页面设计层
- Agent F：04/05 数据与测试层

## 4. 子 Agent 产出边界

子 Agent 必须明确以下内容：

- 负责的文件范围
- 负责的页面组、机制或对象范围
- 不负责的内容
- 依赖的上游基线版本

禁止事项：

- 擅自改动其他 Agent 已锁定的基线字段
- 在未通知主 Agent 的情况下扩展范围
- 用自己的命名规则替代统一口径

## 4.1 推荐任务包模板

主 Agent 分发任务时，建议至少提供：

| task_id | 任务目标 | 负责范围 | 输入基线版本 | 依赖文档 | 输出文件 | 未决问题 | 截止条件 |
|---|---|---|---|---|---|---|---|

子 Agent 回交时，建议至少附带：

- 已修改文件清单
- 引用的基线版本
- 新增假设
- 新增未决问题
- 需要主 Agent 拍板的冲突项

## 5. 汇总规则

主 Agent 汇总时，至少应统一以下内容：

- 节点 ID
- 文档命名
- 术语和字段口径
- 优先级
- 当前版本范围
- 假设与未决问题清单

如果不同子 Agent 产出冲突，处理优先级如下：

1. 统一基线优先
2. 已确认范围优先
3. 主 Agent 汇总口径优先

## 6. 冲突记录规则

出现冲突时，建议至少记录：

| conflict_id | 冲突内容 | 涉及文档 | 决策人 | 最终结论 | 处理时间 |
|---|---|---|---|---|---|

## 7. 最终交付责任

无论使用多少子 Agent，最终交付责任都在主 Agent。

最终交付前，主 Agent 必须确认：

- 三份输入基线已统一
- 各层文档没有互相矛盾
- 强制输出文档清单已满足
- 变更记录和待确认问题已集中汇总

## 8. 适用边界

如果项目规模较小，可以不启用多 Agent 协同。

一旦启用，多 Agent 协同必须服从统一基线和统一命名，不允许每个 Agent 自行定义一套文档体系。
